Turing’s paper does two foundational things at once: it gives a precise model of mechanical computation, and then uses that model to prove that there are well-posed mathematical questions no algorithm can decide in general.

The paper begins by asking what it means for a number to be calculable by a definite procedure. Turing’s answer is to idealize the work of a human calculator into a very simple machine. This machine has a tape divided into squares, a read/write head that examines one square at a time, a finite set of internal states, and a finite table of rules telling it what symbol to print, how to move, and what state to enter next. The importance of this model is not its physical realism but its claim to capture any procedure that is genuinely mechanical and step-by-step.

From this, Turing defines a computable sequence as a sequence of symbols that such a machine can generate, and a computable number as a real number whose digits can be generated by such a process. Much of the early part of the paper is devoted to making this notion exact. He also argues, informally but influentially, that this machine model matches the intuitive idea of effective calculation because a human computer can only inspect finitely much at once, can only carry out finitely many elementary operations, and works according to a finite set of rules.

A key distinction in the paper is between machines that successfully go on producing the relevant output and machines that do not. Turing calls a machine "circular" if it fails to keep producing the sequence in the required way, and "circle-free" if it continues appropriately. This is not yet phrased in the modern language of the halting problem, but it is the same basic phenomenon: there is no general method that, given an arbitrary machine-description, can always determine whether the machine will run in the relevant successful way or fail to do so.

Turing then introduces a way of encoding machines by finite descriptions and numbers. Once machines can be represented as data, one machine can read the description of another and simulate it. This is the paper’s famous universal-machine idea: a single machine can perform any computation whatever, provided it is given the right coded description of the machine to be simulated. This is one of the conceptual origins of the stored-program computer.

The central impossibility argument comes from combining encoding, simulation, and a diagonal-style construction. Turing shows that if there were a general procedure for determining whether an arbitrary machine is circle-free, then one could use it to build a machine that generates a sequence engineered to differ from every sequence in an alleged effective listing of all computable sequences. The construction turns back on itself and yields a contradiction. The conclusion is that there can be no such general decision procedure.

This result is closely connected to the modern halting problem. Turing’s exact formulation is about whether a machine will continue to produce the required output sequence, but the modern restatement is that there is no algorithm that can correctly decide for every program and input whether the program halts. Turing’s paper is one of the original sources of that result.

He then applies this to the Entscheidungsproblem, the decision problem posed by Hilbert: is there a general method that can determine whether any formula of first-order logic is valid or provable? Turing argues that if such a decision procedure existed, then one could use it to settle the machine question that he has just shown unsolvable. Therefore no such general decision method exists. In other words, first-order logical validity is undecidable.

The paper’s lasting significance is fourfold. First, it gives a rigorous analysis of algorithmic procedure. Second, it introduces the universal machine, which becomes the conceptual basis of general-purpose computing. Third, it proves an undecidability result equivalent in spirit to the halting problem. Fourth, it uses that result to show the failure of the Entscheidungsproblem. Together with Church’s work, the paper helped establish the modern theory of computability.

For debate purposes, the most important takeaway is this: Turing does not merely say that some problems are practically hard; he proves that there is no possible general algorithmic method for solving certain classes of problems. The theory of undecidability and the halting-problem framework trace directly back to this argument.
