In *An Introduction to Kolmogorov Complexity and Its Applications*, Li and Vitányi present algorithmic probability and the coding theorem as central links between description length, inductive inference, and universal prediction.

The core setup is usually based on **prefix Kolmogorov complexity**. A binary string’s complexity, often written K(x), is the length of the shortest self-delimiting program for a fixed universal prefix machine that outputs x. The choice of universal machine affects complexity only up to an additive constant; this is the invariance principle. For asymptotic or structural arguments, that constant is treated as machine-dependent but not x-dependent.

From this they define **algorithmic probability**: the probability that a universal prefix machine outputs a given string x when fed an infinite sequence of fair coin flips as program bits. Because programs are prefix-free, the probabilities assigned to outputs form a semimeasure rather than necessarily a full measure. Intuitively, short programs contribute the most probability mass, so strings with many short generating descriptions get higher probability.

A central idea is that this universal a priori probability formalizes **Occam’s razor** and **Epicurus’s principle** together: all computable explanations are allowed, but shorter ones are weighted more heavily. In Solomonoff-style induction, this gives a universal prior over computable environments or outputs.

The book emphasizes that the universal algorithmic probability is **lower semicomputable** but not computable. One can approximate it from below by simulating programs, but in general one cannot compute its exact value because this would solve halting-type problems. This noncomputability parallels the noncomputability of Kolmogorov complexity itself.

The **coding theorem** is the main bridge between probability and complexity. In standard form, it says that the negative logarithm of the universal algorithmic probability of x equals the prefix Kolmogorov complexity of x up to an additive constant:

- K(x) = -log m(x) + O(1)

where m(x) denotes universal algorithmic probability [notation varies across presentations]. The meaning is that strings that are more probable under the universal distribution are exactly those with shorter effective descriptions, up to machine-dependent constants.

The theorem has two complementary interpretations:

1. **From programs to probabilities**: if x has a short program, then x gets relatively large algorithmic probability, because that short program contributes substantial mass.
2. **From probabilities to codes**: if x has high universal probability, then there is an efficient code for x of length about -log m(x), and therefore x must also have low Kolmogorov complexity.

This is why the result is called a coding theorem: it mirrors Shannon-style source coding, but for an individual object under a universal semimeasure rather than a known stochastic source.

Li and Vitányi also connect this to **optimal coding** more generally. Prefix complexity corresponds to ideal code length for individual strings relative to a universal description method. The coding theorem says universal probability induces essentially the same code lengths as shortest effective descriptions. So description length and universal prior are not merely analogous; they are equivalent up to additive constants.

A major use of this framework is in **inductive inference**. Solomonoff prediction mixes over all computable hypotheses, weighted by their simplicity. Li and Vitányi present algorithmic probability as the theoretical gold standard for prediction among computable environments, while stressing that it is incomputable in exact form. This motivates practical approximations such as MDL-style reasoning, compression-based methods, or restricted model classes.

The book also places the coding theorem within the broader architecture of algorithmic information theory:

- **Randomness/incompressibility**: a string is algorithmically random if it lacks substantially shorter descriptions than itself.
- **Structure vs noise**: low-complexity regularities correspond to short programs and thus higher universal probability.
- **Universal prediction**: computable regularities are automatically favored, with a quantitative simplicity bias.

For debate purposes, the main takeaway is that in Li and Vitányi the coding theorem is not a loose heuristic. It is the formal statement that, under a universal prefix machine, the universal prior probability of an object and its shortest effective description length are equivalent up to constants. Any claim that appeals to algorithmic probability as a way of escaping ordinary complexity penalties has to confront this: universal probabilistic weight already tracks compressibility. In that framework, high prior weight comes from short effective generative structure, not from free extra expressive power.

Two caution points the book strongly supports:

- The whole theory depends on **machine-invariant results only up to additive constants**. These constants matter for short strings and fine-grained comparisons.
- The universal prior is **idealized and incomputable**. It is normatively important, but direct implementation is impossible except through approximations or restricted surrogates.

So, as Li and Vitányi present it, algorithmic probability and the coding theorem jointly underwrite a precise simplicity-biased notion of universal induction: shorter computable explanations get more weight, and this weighting is mathematically equivalent to coding by Kolmogorov complexity up to a fixed constant.
